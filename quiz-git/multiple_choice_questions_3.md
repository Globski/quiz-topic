# Question: What is Getting started with Git?

**Answer:** To introduce the basic setup and initial operations required to begin using Git. Creating a first repository and adding and committing files.

---

# Question: What are the key steps in creating your first Git repository?

**Answer:** Initialize a repository, add files, and commit them.

---

# Question: What does it mean to “add” files in Git?

**Answer:** To stage files so they are included in the next commit.

---

# Question: What does “commit” mean in Git?

**Answer:** To save staged changes as a snapshot in the repository history.

---

# Question: What does cloning a repository do?

**Answer:** It creates a local copy of a remote repository.

---

# Question: What is Sharing code?

**Answer:** Methods for sharing repository code with others.

---

# Question: Why is setting user name and email important in Git?

**Answer:** It identifies the author of commits.

---

# Question: What is an upstream remote in Git?

**Answer:** A reference to the original repository that a fork or branch tracks.

---

# Question: What is the purpose of learning about a command in Git?

**Answer:** To understand how to use Git commands and their options.

---

# Question: Why is SSH used with Git?

**Answer:** To securely authenticate and communicate with remote repositories.

---

# Question: What is the purpose of installing Git?

**Answer:** To enable version control functionality on a system.

---

# Question: What is Browsing the history?

**Answer:** To explore and analyze commit history in Git.

---

# Question: What is "Regular" Git Log?

**Answer:** The default output of the `git log` command.

---

# Question: What does the regular Git log display?

**Answer:** A list of commits with details like commit hash, author, date, and message.

---

# Question: What is Prettier log?

**Answer:** Formatting the log output to be more readable.

---

# Question: What is Colorize Logs?

**Answer:** Adding color to log output for better readability.

---

# Question: What is Oneline log?

**Answer:** Displaying each commit as a single line summary.

---

# Question: What is Log search?

**Answer:** Searching through commit history.

---

# Question: What does grouping contributions by author help with?

**Answer:** Understanding contributions from different developers.

---

# Question: What is the purpose of searching commit strings?

**Answer:** To find commits containing specific text.

---

# Question: What is the use of viewing logs for specific lines?

**Answer:** To track changes affecting particular parts of a file.

---

# Question: What is the purpose of filtering logs?

**Answer:** To focus on specific commits based on criteria.

---

# Question: What does “changes inline” mean in git log?

**Answer:** Showing diffs directly within the log output.

---

# Question: What does showing committed files reveal?

**Answer:** Which files were modified in each commit.

---

# Question: What does viewing a single commit include?

**Answer:** The changes, message, and metadata for that commit.

---

# Question: What is the purpose of comparing logs between branches?

**Answer:** To see differences in commit history.

---

# Question: What additional information is included in this one-line log?

**Answer:** Committer name and relative time since the commit.

# Question: What is Working with Remotes?

**Answer:** To manage and interact with remote repositories.

---

# Question: What does deleting a remote branch do?

**Answer:** Removes a branch from the remote repository.

---

# Question: Why would you change a remote URL?

**Answer:** To point to a different repository location.

---

# Question: What does listing remotes show?

**Answer:** All configured remote repositories.

---

# Question: Why remove local copies of deleted remote branches?

**Answer:** To keep the local repository clean and up to date.

---

# Question: What does updating from upstream do?

**Answer:** Synchronizes local changes with the original repository.

---

# Question: What does `ls-remote` do?

**Answer:** Lists references in a remote repository.

---

# Question: What does adding a remote repository enable?

**Answer:** Connecting a local repo to an external repository.

---

# Question: What does setting upstream on a branch do?

**Answer:** Links the branch to a remote branch for push/pull operations.

---

# Question: Why rename a remote?

**Answer:** To improve clarity or match naming conventions.

---

# Question: What type of information is shown about a remote?

**Answer:** URLs and configuration details.

---

# Question: What is the purpose of retrieving a remote URL?

**Answer:** To verify or inspect repository connections.

---

# Question: What is Staging?

**Answer:** To manage how changes are prepared before committing.

---

# Question: What does staging all changes do?

**Answer:** Adds all modified files to the staging area.

---

# Question: What does unstaging a file do?

**Answer:** Removes it from the staging area without discarding changes.

---

# Question: What is a “hunk” in Git?

**Answer:** A portion of changes within a file.

---

# Question: What does interactive add allow?

**Answer:** Selecting specific changes to stage interactively.

---

# Question: What does showing staged changes display?

**Answer:** Differences between staged files and the last commit.

---

# Question: What does staging a single file allow?

**Answer:** Adding only one file to the staging area instead of all changes.
